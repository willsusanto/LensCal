Migration Plan Expo SDK 54 → Next.js App Router PWA

LensCal → Next.js PWA

Full conversion from a React Native Expo app to a Next.js 15 App Router PWA. Login is required; Supabase is the sole data source. SQLite, all native/iOS/Android dependencies, and offline-first sync are removed. Notifications are deferred.

Priority order: UI migration + routing first (Steps 1–9). Settings screen (Step 10) second. Notifications are explicitly deferred (Step 11).

Route Mapping

Every Expo Router screen maps to a Next.js App Router equivalent.

Add user_settings table to Supabase schema

Priority: NOW

Settings were stored in SQLite. In the PWA they live in Supabase so they persist across devices and browsers.

create table if not exists public.user_settings (
  user_id uuid primary key references auth.users(id) on delete cascade,
  default_lens_type text not null default 'monthly',
  monthly_replacement_days integer not null default 28,
  notifications_enabled boolean not null default true,
  reminder_hour integer not null default 8,
  reminder_minute integer not null default 0,
  updated_at timestamptz not null default now()
);
alter table public.user_settings enable row level security;
create policy "Users manage own settings"
  on public.user_settings for all
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);

Add this to supabase/schema.sql. Settings are upserted on first login and on every change.

Risks & Migration Concerns

No data migration path. Existing users who tracked lenses in the Expo app's SQLite DB will lose local data. Since Supabase sync was optional, users who did not sign in will have no cloud copy. This is acceptable if the PWA is a fresh deployment for a new audience.
Offline support is removed. The new app requires a network connection for every read/write. PWA install and service worker caching will serve the app shell offline, but data won't be available without internet.
Supabase RLS schema change. The existing lens_usages partial unique index is per-user (user_id, eye where status = 'active'). Verify the Supabase schema matches before first write, or the one-active-lens constraint won't hold.
Date input UX regression. The native date picker (@react-native-community/datetimepicker) is replaced with <input type="date">. Browser date pickers vary widely in UX. Consider a lightweight library like react-day-picker if the default is unacceptable.
